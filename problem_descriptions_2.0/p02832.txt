### Exercise Title: Sorting Numbers for a Sequence

### Problem Statement:

You have an array of integers, each integer ranging from 1 to N (inclusive), where N is the size of the array. Your task is to perform the minimal number of operations to transform this array into a sequence of numbers from 1 to K (1, 2, 3, ..., K), in any order, where K can be up to N. Each operation consists of removing an integer from the array.

Your goal is to determine the minimum number of integers that need to be removed to achieve this sequence or decide if forming such a sequence is impossible. If it is not possible to form a sequence from 1 to any K using the elements of the array, indicate this.

### Requirements:

1. Implement a function that takes an array of integers where each integer is between 1 and N, inclusive.
2. Determine the minimum number of integers to remove such that the remaining integers can be rearranged into a sequence of consecutive numbers starting at 1.
3. If it is impossible to form any sequence starting from 1 with the given integers, return -1.
4. Optimize the solution to run efficiently for an array size up to 200,000 elements.

This exercise explores the concepts of sequence formation, array manipulation, and greedy algorithms, focusing on efficient resource use and problem-solving skills in determining available patterns in data.
